# Accessibility QA Checklist

- Keyboard can reach skip link, navigation, forms, cookie controls, footer links, and portal upload forms.
- Focus is visible on links, buttons, form fields, and modal controls.
- Cookie preferences dialog can be opened, used, saved, and closed by keyboard.
- Public forms announce missing required fields through native validation or visible error notices.
- Privacy choices form has visible labels for every input.
- Legal pages have one H1 and ordered section headings.
- Text does not overlap at mobile, tablet, and desktop widths.
- Upload notices are readable and linked before file submission controls.
- Images used as decoration have empty alt text; informative images have useful alt text.
