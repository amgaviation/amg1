# Testimonial and Public Proof Release Process

No testimonial, review, case study, client logo, aircraft owner name, crew name, vendor name, tail number, or identifiable aircraft example should be published unless approved.

The hardening migration creates `content_approvals` for future use. Until a UI is built, approvals should be tracked through admin-only database or back-office workflow with these statuses:

- draft
- needs_release
- needs_review
- approved_for_publication
- rejected
- archived

Required review points:

- Name display permission.
- Company display permission.
- Aircraft tail/photo permission.
- Compensation or material connection disclosure review.
- Release document location.
- Legal/brand approval.
